# 验收演讲稿：AI 新闻自动发布系统

---

## 开场（1 分钟）

大家好，我今天分享的项目是 **AI 新闻自动发布系统**。

一句话讲清楚：这是一个会自动更新的 AI 新闻网站。每天跑一条命令，它就从 AI HOT 平台拉取最新的中文 AI 资讯，自动更新到网站，并推送到 GitHub。

不需要编辑，不需要后台，不需要手动排版。**数据进来，网站就更新**。

---

## 演示：看一眼成品（1 分钟）

先给大家看网站长什么样。*（打开 http://127.0.0.1:8080）*

- 左侧：分类导航，按 AI 前沿、产品发布、行业观察、研究论文四个维度筛选
- 中间：时间线信息流，按日期排列，101 条真实新闻
- 顶部：热搜排行榜，Top 3 卡片 + 4-8 名列表
- 每条新闻标题可点击，直接跳转到原文出处

**所有新闻都是真实的**——来自 AI HOT 平台（aihot.virxact.com），不是编的。来源包括 OpenAI 官网、Hacker News、学术论文、行业媒体等。

---

## 系统怎么工作的（2 分钟）

整个系统就三层：

| 层 | 做什么 | 用什么 |
|---|---|---|
| 数据层 | 从 AI HOT 拉数据，转成标准格式 | Python 脚本 `aihot_bridge.py` |
| 展示层 | 纯前端渲染，读 JSON 生成页面 | HTML + CSS + JS |
| 发布层 | 自动 git commit + push | Git |

关键设计：**数据与展示完全分离**。数据存在 `news.json` 一个文件里，前端只管读它来渲染。Agent 只操作这个文件，不碰 HTML 代码。

---

## 核心亮点（3 分钟）

### 1. 真实数据，不是 demo

这不是一个「写死几条假新闻的课程作业」。接入了 AI HOT 的公开 API，每天拉取精选的中文 AI 资讯。目前站上 101 条新闻，覆盖最近一周。每天跑 `python3 aihot_bridge.py --publish` 就自动更新。

### 2. 精确约束

Agent 只改一个文件：`website/data/news.json`。`git add` 硬编码只添加这个文件，Agent 被诱导也改不了别的。架构 + 工具 + Prompt 三层防护。

### 3. 可点击溯源

每条新闻都能点标题跳转到原文。数据是从 AI HOT 过来的，AI HOT 收录了原文链接，我们原样保留。用户看到一条感兴趣的新闻，点一下就回到出处。

### 4. 完整发布链路

```
AI HOT API → aihot_bridge.py → news.json → 网站渲染 → Git push
```

端到端自动化。一个人维护一个 AI 资讯站，每天不到一分钟。

## 现场演示（2 分钟）

演示两个场景：

**场景一：正常同步**
```bash
cd ~/Desktop/dragon-news-agent
python3 aihot_bridge.py --publish
```
→ 拉取最近 7 天精选 → 去重合并 → 写入 news.json → git commit + push → 网站更新

然后刷新网站，看新内容出现在顶部。

## Q&A 备用

**Q: 数据来源可靠吗？**
A: AI HOT 每天从 OpenAI、Hacker News、学术论文、行业博客等源抓取，LLM 生成中文摘要，编辑精选。每条新闻都保留了原文链接。

**Q: 如果 API 挂了怎么办？**
A: news.json 是本地的，网站不依赖 API 实时在线。API 挂了只是不更新，不影响访问。

**Q: 能换成别的数据源吗？**
A: 脚本是解耦的。换个 API，改 `fetch_items` 函数就行，前端不受影响。

**Q: 怎么部署上线？**
A: GitHub Pages 直接托管 `website/` 目录，配合 GitHub Actions 每天跑一次 bridge 脚本，完全免费。
